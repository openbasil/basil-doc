# Diagrams

Source for the site's diagrams. Rendered PNGs live in `../static/images/`.

| Source | Rendered |
| --- | --- |
| `architecture.excalidraw` | `../static/images/architecture.png` |
| `schema-3-authorization-flow.excalidraw` | `../static/images/schema-3-authorization-flow.png` |
| `trust-boundaries.excalidraw` | `../static/images/trust-boundaries.png` |

Some `.excalidraw` sources are generated by a small, deterministic builder (`gen_arch.py`,
`gen_trust_boundaries.py`) so the layout is easy to tweak and re-emit. Edit the builder when one is
present; otherwise edit the Excalidraw source directly, then render.

## Rendering

The `.excalidraw` files render to PNG with the
[excalidraw-diagram skill](https://github.com/) renderer and a Playwright-capable Python.
On this NixOS setup the system `python3` lacks Playwright; use a Nix-store interpreter that
has it (see `docs/agents/excalidraw.md` in the Basil repo for finding one), and always wrap
the render in `xvfb-run -a`:

```sh
# regenerate the .excalidraw from the builder (optional)
python3 gen_arch.py architecture.excalidraw

# render to PNG (PY = a store python3 that can import playwright)
xvfb-run -a "$PY" <skill>/references/render_excalidraw.py architecture.excalidraw --width 1600

# the renderer writes architecture.png next to the source; move it into static/images/
mv architecture.png ../static/images/architecture.png
```
